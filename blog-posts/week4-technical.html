Technical: Hashes and Arrays

Friday, August 29, 2014



    Lately, we have been working a lot with hashes and arrays in Ruby. Hashes are very new to me, and determining which data structure is appropriate in what scenario takes some getting used to.

    An array can be as simple as a list of numbers, or it can be as complicated as lists of lists of lists of objects (that sounds overly complicated, right?). To reference a value in an array, you need to use that values index. Array indices start from 0, so the first item in an array will always have an index of 0, and the last element in an array will have an index of array.length – 1. The #length might look new to you, and if it does, it is a method that returns the length of whatever array you call it on. Arrays can be great for storing things like a list of words to be randomly selected for a game of hang-man, or the details of a person's address.

    Hashes are buddies with arrays, but they can be used very differently. For one, hashes are not bound to strictly numerical indices like arrays, in fact, you can give a hash any index you want. To avoid confusion, hashes don't really use index, they call it a "key." Every key in a hash is matched with a certain value, thus getting us key/value pairs. You might want to store a list of all of your friends and their ages, so you would assign the key as the persons name, and the value as their age. From here you can see a myriad of methods associated with hashes that allow you to play with the data, with things like sorting, deleting and adding methods.

    All I have learned so far in choosing is that it depends. I am even curious if a hash that has values of arrays, maybe that works, that could be pretty useful if you wanted to pull a list of numbers associated with a certain key. Luckily, there is quite a bit of overlap in methods, and they don't change naming conventions all over the place for each class, so you have a lot of repeated methods. Hashes are very cool, and I am glad Ruby has them, as arrays can sometimes be tricky to deal with, at least in the scenarios in which a hash is best.
